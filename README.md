Автоматизированные UI-тесты для Stellar Burgers

Цель проекта
Разработка набора автоматизированных UI-тестов для проверки бизнес-логики frontend-части приложения заказа бургеров. Покрытие тестами критического пользовательского пути: конструктор бургеров, авторизация, оформление заказа и отслеживание в ленте заказов.

Покрытие тестами
Классы Page Object: 5 страниц (MainPage, ConstructorPage, LoginPage, OrdersFeedPage, BasePage)
Сценарии:
- Навигация по приложению (переходы между страницами)
- Работа с конструктором (добавление ингредиентов, drag-and-drop)
- Проверка модальных окон (открытие/закрытие деталей ингредиента)
- Авторизация пользователя
- Оформление заказа и проверка счётчиков
- Отслеживание заказа в ленте («В работе» / «Готовы»)

Технологический стек
Язык: Python 3.8+
Фреймворк: pytest
Инструменты тестирования:
- Selenium WebDriver (для управления браузером)
- Allure (для отчётности и визуализации результатов)
- WebDriverWait + Expected Conditions (для явных ожиданий)
Паттерны проектирования:
- Page Object Model (для разделения логики страниц и тестов)
- Fixtures (для изоляции тестов и переиспользования кода)

Статистика
- Всего тестов: 8 тестовых сценариев (16 прогонов с кроссбраузерностью)
- Процент покрытия основной функциональности: 100%
- Время выполнения: ~2-3 минуты (параллельный запуск в Chrome и Firefox)
- Кроссбраузерность: Chrome + Firefox
Стабильность: 100% (благодаря явным ожиданиям и правильной архитектуре)

Ключевые достижения
- Реализована полная изоляция локаторов от Page Objects 
- Настроена кроссбраузерная параметризация (Chrome + Firefox)
- Создана универсальная фикстура авторизации (login_user)
- Настроена интеграция с Allure для детальной отчётности
- Реализована обработка drag-and-drop для разных браузеров (ActionChains + JavaScript fallback)
